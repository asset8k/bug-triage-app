# LLM Training Logic

This document describes the JSONL training-data generation logic used for
fine-tuned LLM severity classification in CyberTriage.

Primary generator script:
- `dataset/2_convert_to_jsonl.py`

Primary inputs:
- `dataset/train.csv`
- `dataset/test.csv`

Primary outputs:
- `dataset/train.jsonl`
- `dataset/test.jsonl`

---

## Pipeline Overview

```text
dataset/train.csv  ─┐
                     ├─>  dataset/2_convert_to_jsonl.py  ─>  dataset/train.jsonl
dataset/test.csv   ─┘                                    ─>  dataset/test.jsonl
```

The CSV files are prepared by `dataset/1_prep_balance_split_csv.py` using the
four-class severity taxonomy (Critical, Major, Minor, Trivial).

---

## JSONL Record Format (Instruction Tuning)

Each JSONL row is one object with:

| Key | Description |
|-----|-------------|
| `instruction` | Fixed task prompt with output schema and formatting constraints. |
| `input` | Bug report text (trimmed to input limit). |
| `output` | Stringified JSON target the model learns to emit. |

Target output schema:

```json
{
  "severity": "<Critical|Major|Minor|Trivial>",
  "reasoning": "<evidence-based severity rationale>",
  "summary": "<concise one-sentence summary>",
  "description": "<condensed technical description>"
}
```

`severity` is intentionally placed first to prioritize reliable class emission
even when output length is constrained.

---

## Target Construction Strategy

### 1. Summary target
- Derived from the opening section of the bug text.
- Normalized to one clean sentence.
- Avoids trailing punctuation artifacts.

### 2. Description target
- Built as a condensed multi-part slice (beginning, middle, ending context).
- Prefers coverage of technical progression over verbatim copying.
- Avoids ellipsis/truncation patterns that can leak into model output.

### 3. Reasoning target
- Includes a severity rubric cue.
- Extracts compact evidence signals (high-value keywords).
- Ends with a clear severity conclusion sentence.
- Uses light phrase variation to reduce rigid template memorization.

---

## Why This Design

| Decision | Rationale |
|----------|-----------|
| Abstractive targets over extractive copy | Encourages reasoning and compression instead of parroting text. |
| Severity-first JSON ordering | Improves robustness of the primary classification field. |
| Concise evidence mention | Keeps justification technical and focused. |
| Controlled phrase variation | Reduces overfitting to a single sentence template. |
| No truncation markers | Prevents `...` artifacts in generated outputs. |

---

## Model Scope Notes

- The same JSONL design can be used for different fine-tuned LLM families.
- In current CyberTriage runtime, fine-tuned engines include:
  - `llama_finetuned` (Llama GGUF)
  - `qwen_finetuned` (Qwen GGUF)
- This file documents data logic, not model-specific quantization/export steps.

---

## Reproducibility

- Data generation uses deterministic random seed control where randomization is applied.
- Existing JSONL outputs should be regenerated from source CSVs to keep data lineage clear.

---

## Regeneration Command

```bash
cd thesis-bug-triage
python dataset/2_convert_to_jsonl.py
```

Prerequisite:
- `dataset/train.csv` and `dataset/test.csv` exist (generated by `dataset/1_prep_balance_split_csv.py`).
